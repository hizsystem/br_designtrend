# 오늘의 디자인 트렌드 (회사 사업용 버전)

이 폴더는 **일반 대중을 대상으로 하는 회사 사업 아이템**으로서의 디자인 트렌드 리포트 사이트입니다.
디자이너 전용 버전(`../디자인트렌드-매거진/`)을 복제해 만들었고, **두 가지 방향**이 다릅니다.

## 이 버전의 핵심 방향 (반드시 지킬 것)

1. **메인/하이라이트 컬러 = 에너제틱한 주황색** — 이미 `assets/style.css`의 `--accent: #ff6a1a` 등으로 교체됨.
   - 주요 톤: `--accent: #ff6a1a`, 보조 `#ff9f1c`(앰버) `#ff4d6d`(코랄핑크), 그라데이션은 주황→코랄→앰버.
   - 새 요소를 만들 때도 이 주황 팔레트를 유지한다.
2. **누구나 이해하기 쉬운 친절한 말투/용어** — 디자이너 전문용어를 쓰지 않거나, 쓰면 **쉬운 말로 풀어서 설명**한다.
   - 예: "그로테스크 산세리프" → "군더더기 없는 깔끔한 고딕 글씨체", "글래스모피즘" → "반투명 유리 같은 효과",
     "벤토 그리드" → "도시락처럼 칸을 나눈 레이아웃", "키네틱 타이포" → "움직이는 글자".
   - 문장은 짧고 친근하게. 어려운 영어 용어는 괄호로 쉬운 설명을 덧붙인다.

## 폴더 구조

- `index.html` — 홈(히어로 + "오늘 트렌드 보기" 버튼 + 사진 달력으로 지난 호 보기)
- `issues/YYYY-MM-DD.html` — 날짜별 리포트
- `cards/YYYY-MM-DD.html` — 모바일 카드뉴스(좌우로 넘기는 풀스크린 카드)
- `assets/style.css` — 공통 스타일(브랜드 색 한곳에서 관리), `issues.js` — 호 목록(홈·리포트 공유 데이터),
  `report.js` — 리포트 공통 스크립트(왼쪽 목록 + 맨위로 버튼), `cursor.js` — 커스텀 원형 커서
- `PUBLISH-INSTRUCTIONS.md` — 리포트 1편을 만드는 상세 절차(구조·규칙). **친절체로 작성**할 것.

## 리포트 1편 구조 (디자이너 버전과 동일한 뼈대, 톤만 친절하게)

- 5개 카테고리(친절한 이름 권장):
  - 타이포그래피 → "글씨(폰트) 트렌드"
  - 메인비주얼 그래픽 → "이미지·그래픽 트렌드"
  - 웹 & 앱 디자인 → "웹·앱 화면 트렌드"
  - 컬러 조합 → "색 조합 트렌드"
  - 카드뉴스 → "카드뉴스 트렌드"
- 각 카테고리 3블록: ① 지금 주목받는 것 ② 사람들의 반응 ③ 👉 이렇게 써보세요(참고 포인트)
- 글씨 트렌드엔 "지금 뜨는 폰트"를 **제목용/본문용**으로 나눠 미리보기(무료 폰트는 실제 글꼴로 렌더, 유료는 제공처 링크).
- 모든 예시 이미지엔 **출처 캡션**(회색 작은 링크) 필수. 보여주는 이미지는 링크 클릭 시 보이는 실제 작업과 일치.
- ①②③ 번호 나열은 `<ul class="enum">`로 **번호마다 한 줄씩**.
- 요약 대제목은 의미 단위로 `<br>` 줄바꿈, 날짜·부가 소제목 금지.
